두 단계의 파이프라인

Research Helper는 크게 두 개의 자동화 파이프라인으로 구성되어 있습니다. 먼저 주제 선정 파이프라인이 코퍼스에서 분석할 가치가 있는 주제를 자동으로 골라내고, 사용자가 그 중 하나를 선택하면 보고서 생성 파이프라인이 목차 기반의 심층 보고서를 작성합니다.

PART 1 — 주제 선정

최근 30일 코퍼스 → 핵심 주제 5~10개

매일 빌드되는 아카이브에서 최근 30일 기사를 모아 키워드로 1차 필터링한 뒤, GLM-4.7 thinking 모드 LLM이 기사를 통째로 읽고 의미 있는 주제를 식별합니다. 식별된 각 주제는 한 번 더 전체 아카이브에서 보강 기사를 찾아 풍부하게 만듭니다.

03PASS 1
GLM-4.7 thinking · 1 call

LLM이 코퍼스 전체를 읽고 후보 주제 식별

필터링된 기사를 한꺼번에 LLM에 입력합니다. LLM은 thinking 모드에서 코퍼스를 통독한 뒤, "키워드 일치"가 아니라 "같은 시장 현상을 가리키는 기사 묶음"을 의미적으로 클러스터링하여 5~10개의 후보 주제를 출력합니다.

핵심 원리. "Amazon이 Globalstar를 인수했다"와 "위성통신 시장 변곡점 분석"은 키워드를 공유하지 않지만 D2D 위성 연결성으로의 구조 전환이라는 동일 현상이므로 한 주제로 묶입니다. 반대로 단순 정기 데이터 업데이트나 방향성 없는 일상 트래커 보고는 제외하도록 지시받습니다.
📊 실제 예시
"메모리 가격 폭등"이 자동으로 묶임
  • 📍 Counterpoint — Q1 출하 -6% YoY, 메모리 부족
  • 📍 Omdia — Memory cost · macro volatility weigh on shipments
  • 📍 DigiTimes — 'Chipflation' hits Samsung, boosts Apple
  • 📍 TrendForce — Samsung 메모리 영업이익 +755%

4개 기관이 다른 단어로 같은 사건을 보도 → 하나의 후보 주제로 묶임.

04CRIT
자동 분류

각 후보 주제에 Criterion 라벨링

LLM은 각 주제를 다음 두 기준 중 하나(또는 둘 다)로 분류합니다. 기준 자체는 후보 개수를 강제하지 않으며, 풀에 신호가 약하면 5개로 줄고 풍부하면 10개까지 늘어납니다.

Criterion 2 — Hot
다중 출처 합의 신호

서로 다른 2개 이상의 기관이 같은 시장 현상을 30일 안에 보도. 키워드가 달라도 의미가 같으면 같은 현상으로 인정합니다.

Criterion 3 — New
새롭게 등장한 신호

30일 안에 처음 등장한 의미 있는 단일 출처 신호. 특정 OEM의 시장 진입·전략 전환·가시화된 경쟁 우위 등.

두 기준을 동시에 만족하면 Criterion 2+3 (Hot · New)으로 표시됩니다. UI에서는 각각 노랑/초록/보라 배지로 구분됩니다.
📊 실제 예시
"Chipflation" 토픽 라벨
  • 4개 기관 합의 ✅ Criterion 2 인정
  • Tracker 관점 (출하 -6%) — Counterpoint·Omdia·IDC
  • 공급망 관점 (Samsung 메모리 +755%) — DigiTimes·TrendForce
  • 같은 사건의 양면 ⚡ Cross-layer 강한 신호

출하 감소(시장)와 공급망 leak(기업)이 동시에 잡힌 가장 강력한 패턴.

05ENRICH
GLM-4.7 thinking · N calls

각 주제에 대해 전체 아카이브 재검색 → 보강

Pass 1에서 식별된 N개 주제 각각에 대해, 전체 아카이브를 다시 검색하여 추가 기사를 찾습니다. 그런 다음 LLM이 원래 인용한 기사 + 새 기사를 모두 보고 institution_count, key_data, rationale을 갱신합니다.

결과적으로 각 주제는 다음 필드를 갖춥니다:
  • title — 한국어 명사구 제목
  • criteria — Criterion 2 / 3 / 2+3
  • institution_count — 인용된 고유 기관 수
  • articles — 인용 기사 목록 (날짜·출처·제목)
  • key_data — 정량 지표 (%, $, 출하량 등)
  • rationale — 왜 구조적 신호인지 2~3문장 (한국어)
저장 위치: scripts/_topic_suggestions.json. 랜딩 페이지의 추천 주제 카드는 GET /api/topics/suggested로 이 결과를 읽어 보여줍니다.

AQUERY
LLM · Korean → English

한국어 주제를 영문 검색 쿼리로 변환

"위성 통신의 대중화" 같은 한국어 자연어 주제를 받아, Tier-1 영문 매체에서 가장 잘 잡히는 검색 쿼리(키워드 조합)를 LLM이 만들어냅니다. 이후 모든 검색은 이 쿼리를 기반으로 합니다.

BSEARCH
3-tier · archive first

아카이브 + RSS + DDG로 1차 검색

먼저 로컬 아카이브 JSON에서 토픽 관련 기사를 찾고, 부족하면 RSS 피드를, 그래도 부족하면 DuckDuckGo로 외부 검색까지 자동 fallback합니다. 결과는 본문 fetch 가능 여부에 따라 분기됩니다.

📊 실제 예시
약 50건 발견

CTOC
LLM · table of contents

목차(TOC)와 섹션별 영문 검색어 생성

1차 검색 결과를 컨텍스트로 LLM이 보고서 목차를 짭니다. 각 섹션마다 어떤 영문 키워드 조합으로 추가 검색을 돌릴지도 함께 정해집니다. 일반적으로 4~6개 섹션 + 시사점.

📊 실제 예시
3개 섹션 목차
1. 메모리 부족이 만든 시장 양극화 (원인)
2. Apple·Samsung의 마진 방어 vs ODM·중저가의 위기 (영향)
3. 인도·중국 시장의 하방 압력 (결과)
GATE 1 사용자 검수. 생성된 목차와 검색어를 UI에서 확인·수정. --auto 시 자동 통과.
DDEEP
per-section search

섹션별 정밀 검색

확정된 목차의 각 섹션에 대해 다시 한 번 3-tier 검색을 돌려 더 풍부한 자료를 모읍니다. 이 단계에서 인용 가능한 본문(fetchable) 기사는 본문을 가져와 캐싱합니다.

📊 실제 예시
섹션별 자동 추출 검색어
  • 섹션 1: "DRAM price surge 2026" + "memory crunch shipment"
  • 섹션 2: "Apple Q1 2026 record revenue" + "Samsung memory profit"
  • 섹션 3: "India smartphone 6 year low" + "China memory price hike"

3개 섹션 합산 약 60건 추가 수집 → 총 110건.

GATE 2 사용자 검수. 섹션별 검색 결과 확인. 자료가 부족한 섹션은 통과시키지 않을 수도 있음.
E·FWRITE
LLM · per-section

섹션별 분석 + 본문 작성

각 섹션의 검색 결과를 컨텍스트로 LLM이 본문을 작성합니다. 인용은 [N] 형식으로 자동 번호가 붙고, 출처 URL이 보고서 말미에 일괄 정리됩니다.

📊 섹션 1 본문 (일부)
한국어 본문 자동 작성
"2026년 1분기 글로벌 스마트폰 시장은 DRAM 및 CPU 부족으로 인한 부품 공급망 위기와 중국 시장의 위축이라는 이중고를 겪었습니다. 그러나 Apple은 iPhone 17의 강력한 sell-through 성과로..."

섹션당 10~25초, 3섹션 총 약 50초.

데이터 소스

7개 Tier-1 기관 · 4개 관점 레이어

모든 출처는 카테고리(A·C·E·F)로 태깅되어 있습니다. 같은 현상을 서로 다른 레이어에서 보도하면 "교차 레이어 보강(cross-layer corroboration)"으로 인식되어 더 강한 신호로 평가됩니다.

#기관카테고리역할
01Counterpoint ResearchA · Tracker출하·점유율 정량 데이터의 핵심 트래커
02TrendForceA · Tracker대만 기반 메모리·디스플레이 출하 데이터
03OmdiaA · Tracker디바이스·디스플레이 시장 추적
04IDCA · Tracker글로벌 출하·예측의 베이스라인
05Yole GroupC · Component반도체 패키징·광학 부품 전문
06DigiTimes AsiaE · Supply대만·한국·중국 ODM·EMS 공급망 leak
07CCS InsightF · CarrierUK·EU 통신사·소비자 행동 관점
Cross-layer 예시. DigiTimes가 'Chipflation' 공급망 leak을 보도하고 같은 시기에 Counterpoint·Omdia가 메모리 가격 상승에 따른 출하량 감소를 보도하면, 이는 단순한 두 기관 합의가 아니라 "공급망 압박 → 트래커 데이터 확인"의 supply-side와 demand-side가 양면에서 확정된 strong signal입니다. LLM은 이런 교차 레이어 corroboration을 우선 가중합니다.
