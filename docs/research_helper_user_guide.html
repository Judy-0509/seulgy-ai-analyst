First-time guide

처음 보는 사람도 이해하는 리서치 자동화 툴

Research Helper는 스마트폰과 휴머노이드 시장의 Tier-1 아카이브를 모아, 핵심 주제와 새롭게 떠오르는 주제를 자동으로 고르고, 사용자가 검토할 수 있는 단계형 보고서 작성 흐름으로 연결하는 도구입니다.

한 문장으로 말하면

  • 무엇을 볼지 최신 아카이브에서 분석 주제를 추천합니다.
  • 왜 중요한지 여러 출처와 시장 레이어를 연결해 판단합니다.
  • 어떻게 쓸지 목차, 검색, 본문 분석, 시사점까지 보고서로 만듭니다.

1. 이 툴은 무엇을 하는가

추천 주제 탐색 + 보고서 생성 + 근거 관리

아카이브 기반 탐색

각 리서치 기관과 전문 매체의 기사/리포트 메타데이터를 JSON 아카이브로 모읍니다. 이후 검색은 웹 전체가 아니라, 먼저 검증된 아카이브에서 시작합니다.

2. 처음 사용할 때의 화면 흐름

랜딩 화면에서 시작해 리포트 아카이브까지

도메인을 고릅니다

왼쪽 메뉴에서 Smartphone, Humanoid 등 관심 도메인을 선택합니다. 도메인을 바꾸면 랜딩 화면의 배경, 추천 주제, 키워드 기준이 함께 바뀝니다.

추천 주제를 확인합니다

랜딩 화면에는 핵심 주제와 새롭게 등장한 주제가 나뉘어 표시됩니다. 각 주제를 펼치면 근거 기사, 주요 데이터, 선정 이유를 볼 수 있습니다.

이미 작성된 리포트가 있으면 바로 봅니다

추천 주제에 연결된 리포트가 있으면 Archive 상세 화면으로 이동합니다. 오른쪽 참고 수치 패널에서 본문에 쓰인 근거와 출처를 확인할 수 있습니다.

새 분석이 필요하면 보고서 생성을 시작합니다

로그인 후 /app에서 주제를 입력하거나 추천 주제를 선택합니다. 이후 8단계 파이프라인이 실시간으로 진행됩니다.

개발 실행 예시
python start.py

주요 화면
http://localhost:5173/          추천 주제 랜딩
http://localhost:5173/app       보고서 생성
http://localhost:5173/archive   보고서 아카이브
http://localhost:5173/db        소스별 아카이브 DB
http://localhost:5173/keywords  도메인별 키워드

3. 핵심 주제와 새로운 주제를 뽑는 방식

메이저 패스와 Curiosity 패스

핵심 주제: 30일 메이저 패스

최근 30일 아카이브를 읽고, 여러 출처가 같은 현상을 서로 다른 관점에서 확인하는지를 봅니다. 예를 들어 출하 데이터, 공급망 보도, 부품 분석이 같은 방향을 가리키면 핵심 주제가 됩니다.

다중 출처 시장 영향 반복 신호

새로운 주제: 7일 Curiosity 패스

최근 7일 안에 등장한 작지만 흥미로운 움직임을 찾습니다. 아직 대세는 아니지만 마이너 플레이어의 전략 변화, 역방향 데이터, 단발 기술 fact, 대형 기업의 비주류 행동을 포착합니다.

niche contrarian off-trend

주제 선정의 판단 기준

기준 무엇을 보는가 예시
출처 다양성 한 기관의 단독 주장인지, 여러 독립 출처가 같은 현상을 말하는지 확인합니다. Counterpoint 출하 데이터 + DigiTimes 공급망 보도
레이어 교차 시장 데이터, 공급망, 부품, 소비자/채널 관점이 서로 연결되는지 봅니다. 부품 부족이 출하량 감소와 가격 전략 변화로 이어지는 경우
시장 영향 단순 뉴스가 아니라 점유율, 출하, 가격, 경쟁 구도, 도입 속도에 영향을 주는지 판단합니다. AI 데이터센터 수요가 스마트폰 메모리 공급을 압박
새로움 기존 리포트와 중복되는 주제는 줄이고, 최근 새로 등장한 신호를 분리합니다. 신규 휴머노이드 업체의 첫 상용 계약

핵심 주제는 “충분히 강한 시장 현상”을 찾는 과정이고, 새로운 주제는 “아직 작지만 애널리스트가 눈여겨볼 만한 신호”를 찾는 과정입니다. 그래서 두 패스는 서로 다른 시간 창과 판단 기준을 사용합니다.

4. 선택한 주제로 리포트가 작성되는 과정

쿼리 생성부터 저장까지 8단계

01
영문 쿼리 생성 한국어 주제를 검색 가능한 영어 쿼리로 바꿉니다.
02
아카이브 사전 검색 기관별 JSON 아카이브에서 관련 기사 메타데이터를 찾습니다.
03
목차 생성 검색 결과를 바탕으로 보고서의 섹션과 분석 각도를 설계합니다.
04
GATE 1 사용자가 목차와 섹션별 검색어를 확인하고 확정합니다.
05
섹션별 검색 확정된 목차마다 필요한 근거를 다시 좁혀 검색합니다.
06
GATE 2 섹션별 검색 결과를 보고, 더 검색할지 분석으로 갈지 결정합니다.
07
본문 분석 본문 fetch가 가능한 자료는 읽고, 섹션별 분석 문단을 작성합니다.
08
시사점과 저장 Executive Summary, 시사점, 참고 수치를 구성해 리포트로 저장합니다.

보고서에서 확인할 수 있는 것

본문 리포트

Executive Summary, 조사 배경, 핵심 분석 섹션, Market Insights를 확인합니다. 섹션별로 근거가 되는 수치와 출처가 연결됩니다.

오른쪽 참고 수치 패널

본문에 사용된 링크와 근거를 카드로 보여줍니다. 현재 보고서가 스마트폰이어도 자동차 또는 휴머노이드 자료를 참고했다면 해당 도메인의 색상으로 표시됩니다.

5. 처음 보는 사람이 알아두면 좋은 용어

화면과 파이프라인에 자주 나오는 표현

용어 뜻
Archive 기관별 기사/리포트 메타데이터를 저장한 로컬 JSON 데이터베이스입니다.
Tier-1 source 주제 선정과 보고서 생성에서 우선적으로 신뢰하는 리서치 기관, 전문 매체, OEM 발표 소스입니다.
Criterion 2 여러 출처가 같은 시장 현상을 확인하는 핵심 주제 성격입니다.
Criterion 3 최근 새롭게 등장한 단발 신호, contrarian 신호, niche 움직임입니다.
GATE AI가 다음 단계로 넘어가기 전에 사용자가 목차 또는 검색 결과를 확인하는 단계입니다.
Custom view 보고서 내용을 간결한 브리핑 형태로 다시 보여주는 보기 모드입니다.
